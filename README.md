# Image Flow — gpt-image-2 ノードエディタ (GitHub Pages版)

ブラウザ上でノードをつないで gpt-image-2 の画像生成・編集ができるツールです。
このバージョンはサーバー不要で、GitHub Pages にそのまま公開できます。

- APIキーはページ右上の「APIキー」から入力し、**使う人のブラウザ内(localStorage)にだけ**保存されます
- キーの送信先は api.openai.com のみ。GitHubにもどこにもキーは置かれません
- なので、このリポジトリはPublicにしても安全です

## 公開手順(初回のみ・PCにNode.jsやGitは不要)

1. GitHubにログインし、右上「＋」→「New repository」でリポジトリを作る
   - 名前は好きに(例: `image-flow`)。Public を選択
2. 作ったリポジトリのページで「uploading an existing file」をクリックし、
   このフォルダの**中身を全部**ドラッグ&ドロップして「Commit changes」
   - `.github` フォルダも忘れずに(隠しフォルダなので表示設定に注意)
3. リポジトリの **Settings → Pages** を開き、Source を「**GitHub Actions**」にする
4. リポジトリの **Actions** タブで緑のチェックがつくまで1〜2分待つ
5. Settings → Pages に表示されるURLを開く:
   `https://ユーザー名.github.io/リポジトリ名/`

以降このURLをブックマークしておけば、いつでもどこからでも使えます。

## 使い方

1. 右上の「APIキーを設定」に自分のOpenAI APIキー(sk-...)を入力して保存
2. ノードをつないで生成:
   - **＋ プロンプト** — テキスト入力ノード
   - **＋ 画像** — 参照画像ノード(クリックまたはドロップで読み込み)
   - **＋ 生成ノード** — gpt-image-2 を呼び出すノード
3. ノードの右端の○から生成ノードの左端の○へドラッグして接続し、「生成」
   - プロンプトのみ → 新規生成 / 画像もつなぐと → 画像編集・合成(複数枚OK)
   - 生成結果を別の生成ノードへつなぐチェーンも可能
   - アスペクト比・品質・枚数(1〜4)は生成ノード内で指定
4. 線はホバーで出るハサミで切断、ノードは選択して Backspace で削除

## 更新のしかた

修正版のファイルを受け取ったら、GitHubのリポジトリページで
「Add file → Upload files」から同じ場所に上書きアップロードしてコミットするだけ。
1〜2分後にページをリロードすれば反映されます(URLは変わりません)。

## 注意

- 複雑なプロンプトは生成に最大2分ほどかかることがあります
- gpt-image-2 は透過背景に非対応です
- 生成はトークン課金です。品質 low で試してから high に上げるのがおすすめ
- 共有PCで使ったときは、APIキーのパネルから「削除」を押してください
